# gTOKEN

## Intro

### The Law of gTOKEN – True Games Tokenization Philosophy

- g = game
- gTOKEN = game TOKEN
- game TOKEN = a token in a game

**Core Law of gTOKEN:**

- If a game is truly tokenized, then if you remove any gTOKEN from the equation the game should no longer function.
- If you can remove the token features/integration from the game and it still functions, it is not a tokenized game, it is just uses cryptocurrencies.

**The Law Applied to Tower Defense Example:**

In the 3v3 tower defense (3 lanes, one fortress):

- Score itself = gTOKEN
- Units are spawned every 60s by paying gTOKEN cost
- Players can choose any lane (strategy)
- Buildings can be constructed at any time paying gTOKEN cost
- Units killed by towers reward gTOKEN
- If the fortress kills do not reward gTOKEN, there is effectively a burn on the gTOKEN (cost gTOKEN to send units but no gTOKEN rewarded)
- Without the SCORE in the game, there is no game
- Everything changes once you tokenize the SCORE into a token because the entire game is now tokenized and all the interactions now have economical impact on the tokenomics

**The Law Applied to StarCraft-style Resource Example:**

In StarCraft FFA with 6 players:

- All 6 players pool gTOKEN together equally
- gTOKENs are what minerals do in the current game
- Map is randomly created and gTOKEN is spread across map
- Now it is a real battle with real value on the line and it exchanges hands in real time onchain
- The resource is a gTOKEN and without the RESOURCE in the game, the game would also break

**SCORE/RESOURCE as gTOKEN Comparison:**

- For RESOURCE you could just replace gTOKEN with USDC (maybe minerals = USDC and gas = SOL because it’s volatile gas).
- It’s effectively the same thing since all tokens are atomically swappable to any other tokens.
- But you can’t do much with USDC and SOL except wager with it.
- While with gTOKEN you can add layers of mechanics that add depth.
- Unlike SCORE, in the RESOURCE example: if the RESOURCE is simply nothing more than just a currency, it can just be nothing more than a stable currency like USDC.
- But both game formats respect the law of gTOKEN — the game breaks without the gTOKEN.


## How gTOKEN Is Used in Tower Defense

**gTOKEN – Structured Summary**

### 1. Core Definition & Purpose

- gTOKEN is explicitly the **utility token** and the **game token**.
- It serves as the single universal component that connects the original program and every forked version of the game.
- It directly replaces what would traditionally be a "score" in the game.

### 2. How gTOKEN Works In-Game (Mechanics)

- Every unit deployed costs gTOKEN.
- Every building or tower built costs gTOKEN.
- Every kill earns gTOKEN.
- Every destruction earns gTOKEN.
- Players can optionally wager or bet additional gTOKEN before a match begins.
- The winner of the match claims the entire wagered gTOKEN pool.
- After wagering closes, RNG determines any modifiers.
- It always costs gTOKEN to play a match, but a winner is expected to net earn gTOKEN overall by killing more units than they lose.

### 3. Token Design & Economic Model

- gTOKEN is launched on a **flatish curve**.
- The curve is priced so that a basic game costs roughly ~$1, while players can add extra gTOKEN on top for larger wagers.
- Every match played causes:
  - Token supply to expand organically.
  - Liquidity pool to expand organically.
  - Price to gradually increase due to the curve.
- The liquidity pool itself generates fees that go to the game.
- In addition, the game collects a small fee of 1 to 10 basis points in native tokens on every onchain action.
- Over time this builds a treasury consisting of both stablecoins and gTOKEN.
- Speculation combined with genuine gameplay activity further fuels market participation and pool growth.

### 4. TGE & Launch Mechanism

- TGE is driven **purely through organic gameplay**.
- Players buy gTOKEN to participate, and those purchases directly fill the liquidity pool.
- The same pool immediately serves as the exit liquidity for anyone to cash out gTOKEN back into stablecoins or whatever base asset is used.
- Once the number of active users exceeds the initial flat-ish curve supply, TGE is considered essentially complete.
- At that point the token enters **full price discovery**, where the market sets the price of the utility token.
- gTOKEN is designed to power both the original program and every player-created fork indefinitely.
- If the game never gains any play, the tokens simply remain on the curve forever, described as harmless "magical internet tokens doing nothing" rather than locking real money in useless pools.

### 5. Forking System & Long-Term Incentives

- gTOKEN stays the shared, universal token across **all forks** of the program.
- Any fork can adjust its own fees however it chooses.
- Each fork has every incentive to become the best version possible to attract players.
- As long as gTOKEN continues to be adopted through these forks, overall activity flows back to gTOKEN.
- That increased activity ultimately drives value back to the original program.
- Early-snipe scenario: anyone could buy the entire supply at launch and "snipe it," in which case they would effectively own the game.
- Counter-balance: if that holder charges too much or makes players unhappy, users can simply fork the program and create a new gTOKEN.
